Grid vs Flex: entendendo a diferença na prática

Este documento HTML aplica, ao vivo, exatamente os conceitos que ele está explicando.

A plataforma do curso ensina que, pra dispor os cards desse jeito, se adaptando ao layout, o caminho é usar display: grid — e na prática é mesmo a ferramenta mais adequada aqui. Mas vale entender por quê: dá pra chegar num visual parecido com Flexbox também, então a diferença real está em como cada um se comporta nos detalhes, principalmente quando o número de itens não fecha uma linha exata.

1. O mesmo visual, dois jeitos diferentes de chegar lá

Mexa nos controles abaixo. Os dois painéis usam exatamente os mesmos cards, com a mesma largura mínima e o mesmo espaçamento — um construído com Flexbox, o outro com Grid.

Flexbox
Grid

Repare: com a maioria das combinações, os dois ficam visualmente idênticos. Mas experimente um número de cards que não feche a última linha exatamente (tipo 7 ou 9) — olha o que acontece com o tamanho dos cards da última fileira em cada um.

No Flexbox

  • Os itens da última linha incompleta esticam pra preencher o espaço sobrando, porque flex-grow distribui o espaço livre entre os itens daquela linha.
  • Isso quebra o alinhamento visual em coluna entre as linhas — cada linha "decide" sozinha como dividir o espaço.

No Grid

  • As colunas são definidas para a grade inteira, não linha por linha — então a última linha incompleta simplesmente deixa espaço vazio, sem esticar os itens.
  • Os cards continuam alinhados em coluna entre todas as linhas, porque a grade é uma estrutura única, não fileiras independentes.

2. O código por trás de cada um

Isto é literalmente o CSS que está rodando nos dois painéis acima:

/* Flexbox */
.cards-flex {
  display: flex;
  flex-wrap: wrap;
  gap: 12px;
}

.cards-flex .card-item {
  flex: 1 1 90px; /* cresce, encolhe, base de 90px */
}
/* Grid */
.cards-grid {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(90px, 1fr));
  gap: 12px;
}

Repare que o Flexbox precisa da regra flex: 1 1 90px no item pra crescer — é aí que mora o comportamento de "esticar a última linha". O Grid resolve isso inteiramente na definição do container, com minmax(), sem nenhuma regra adicional no item.

3. Então quando usar cada um?

A afirmação certa não é "grid é obrigatório", é "grid é a ferramenta certa quando você quer controlar duas dimensões ao mesmo tempo (linha e coluna) como uma estrutura única". Flexbox continua sendo a ferramenta certa quando você só precisa organizar itens numa direção (uma barra de navegação, uma lista de botões, um conjunto de tags).

Pro caso específico de photo-grid e places do projeto — onde o requisito é alinhamento consistente em colunas, com itens de tamanhos variados que precisam se comportar como uma grade de verdade — Grid é a escolha mais adequada, não a única tecnicamente possível. É uma diferença pequena de palavras com uma diferença grande de entendimento.
