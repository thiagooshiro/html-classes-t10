Mas afinal, o que é BEM?

Uma metodologia de nomenclatura CSS. Esse HTML usa BEM pra se explicar.

Block

Eu sou um Bloco.

Minha classe CSS é .explainer. Um Block é uma unidade independente — faz sentido sozinho, sem depender do contexto ao redor. Pense nele como um componente: um card, um menu, um formulário, um header.

Um bloco tem um nome simples, sem underscores nem hífens duplos. Tudo que está dentro de mim pertence a mim — e vai dizer isso no próprio nome da classe.

.explainer { ... } /* ← isso é um bloco */
Element

Eu sou um Título. Mas não sou qualquer título.

Minha classe é .explainer__title. Esse __ (dois underscores) é a assinatura de um Element — um filho que pertence a um bloco específico.

Um Element não existe fora do seu bloco. O .explainer__title é o título desse componente. Se ele aparecer em outro lugar, ele vai ter outra classe — porque vai pertencer a outro bloco.

.explainer__title { ... } /* ← bloco__elemento */
Modifier

E quando um bloco precisa de uma variação?

Entra o Modifier — marcado com -- (dois hífens). Ele descreve uma variação de estado ou aparência de um bloco ou elemento.

Os dois cards abaixo são o mesmo bloco .status-card. A diferença está no modifier:

✅

status-card--success

Tô verde porque tenho o modifier --success.

⚠️

status-card--warning

Tô laranja porque tenho o modifier --warning. Mesmo bloco, outra personalidade.

❌

status-card--error

E eu sou o --error. Ninguém me confunde com os outros dois.

.status-card--success { ... } /* ← bloco--modifier */

Você acabou de ler BEM aplicado.

Todo esse HTML usa a metodologia BEM — blocos, elementos e modificadores — na nomenclatura das próprias classes que explicam o conceito.

A lógica é simples: bloco é o componente, bloco__elemento é uma parte interna, e bloco--modifier é uma variação.

Guarda esse arquivo. Ele é seu guia de referência rápida pra qualquer projeto que vier pela frente.

.bloco + .bloco__elemento + .bloco--modifier
