position: static relative absolute fixed sticky

01

static

Valor padrão. O elemento segue o fluxo normal do documento. Propriedades top/left/right/bottom e z-index não funcionam.

playground — elementos estáticos
A
B
C
D
Todos com position: static. Eles aparecem na ordem do HTML. Não é possível movê-los com top/left.
.elemento {
  position: static;
}

02

relative

O elemento continua ocupando seu espaço no fluxo, mas pode ser deslocado com top/left em relação à sua posição original.

playground — mova o elemento B
A
B
C
B está no lugar normal. O espaço dele está reservado.
.B {
  position: relative;
  top: 0; left: 0;
}

03

absolute

O elemento sai do fluxo e se posiciona em relação ao ancestral mais próximo com position diferente de static. Se não existir, usa o <html> (viewport).

playground — toggle no pai (.card)
.card — position: static
Conteúdo do card
NOVO
O .card está com position: static. O badge olha para o pai, vê static, então sobe até a página (viewport) e fica no canto superior direito da tela.
.card { position: static; }
.badge { position: absolute; top: 10px; right: 10px; }

04

fixed

Sai do fluxo e se ancora na viewport (janela do navegador). Não rola com a página. A própria navbar deste site usa position: fixed.

playground — simulação de um modal/botão fixo
📌 position: fixed
O quadrado laranja tem position: fixed; bottom: 20px; right: 20px; em relação à viewport. Role a página — ele continua fixo.
.fixed-box {
  position: fixed;
  bottom: 20px; right: 20px;
}

05

sticky

Híbrido: age como relative dentro do container, mas "gruda" quando atinge o limite (top: 0) enquanto o pai está visível.
