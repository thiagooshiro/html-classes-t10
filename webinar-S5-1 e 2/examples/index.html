Blocos CSS — display

Todo elemento HTML tem um comportamento padrão de layout. Esse comportamento é controlado pela propriedade display. Entender os três valores principais é fundamental antes de qualquer coisa.

display: block

Ocupa a largura inteira disponível. Empurra o próximo elemento pra baixo. <div>, <p>, <h1>–<h6>, <section>, <header>, <footer> são block por padrão.

display: inline

Fica na mesma linha. Não aceita width nem height. O padding e margin horizontais funcionam, mas os verticais não empurram outros elementos. <span>, <a>, <strong> são inline por padrão.

display: inline-block

Fica na mesma linha e aceita width, height, padding e margin. O melhor dos dois mundos quando você quer elementos lado a lado com tamanho controlado.

⚠ armadilha clássica
Elementos inline ignoram width e height. Se você colocar essas propriedades num <span> sem mudar o display, não vai acontecer nada e você vai passar 40 minutos sem entender por quê.

Box Model

Todo elemento block é uma caixa formada por quatro camadas. Entender onde cada uma termina e começa é o que separa quem "vai no chute" de quem controla o layout de verdade.

Box model visual
margin
padding
content
margin border padding content
Exemplo
.card {
  width: 300px;        /* largura do content */
  padding: 20px;       /* espaço interno */
  border: 2px solid #000;
  margin: 16px 0;       /* espaço externo */
}

/* sem box-sizing: largura real = 300 + 40 + 4 = 344px */
/* com box-sizing: border-box → largura real = 300px */

box-sizing: border-box

Com border-box, o width que você define já inclui padding e border. Sem ele, o elemento cresce além do que você esperava. O checklist do P5 tolera apenas 30px de diferença em 1280px — sem border-box você vai errar isso.

Reset recomendado
*,
*::before,
*::after {
  box-sizing: border-box;
}
⚠ armadilha clássica
O checklist rejeita o uso de margin negativo. Se você está usando margin negativo pra corrigir algum desalinhamento, é sinal de que o problema está em outro lugar — geralmente no box model ou em padding acumulado.
pergunta pra reflexão

O design original mostra uma seção com 24px de espaço entre o texto e a borda. Você coloca padding: 24px, mas o elemento fica maior do que no design. O que provavelmente está faltando?

Unidades CSS

Escolher a unidade errada é uma das causas mais comuns de layout quebrado em tamanhos de tela diferentes. Aqui estão as que você vai usar no P5:

px
Pixel absoluto. Fixo, não escala.
border: 1px solid
%
Relativo ao elemento pai.
width: 100%
rem
Relativo ao font-size do <html>. Geralmente 16px.
font-size: 1.5rem
em
Relativo ao font-size do elemento pai. Cuidado com encadeamento.
padding: 1.2em
vw
1% da largura da viewport. Útil pra tipografia fluida.
font-size: 5vw
vh
1% da altura da viewport. Útil pra seções full-screen.
height: 100vh
💡 regra prática
Bordas e sombras → px. Fontes → rem. Larguras de container → % ou px com max-width. Altura de elementos de texto → nunca fixe em px (o checklist rejeita isso).
⚠ armadilha do checklist
O P5 rejeita explicitamente elementos de texto com altura fixa. Nunca coloque height: 40px num parágrafo ou título — o texto vai vazar quando o conteúdo crescer.
04 /

Responsividade

Layout responsivo significa que o design se adapta a diferentes tamanhos de tela sem quebrar. No P5, você vai precisar que tudo funcione corretamente em três breakpoints: 1280px, 768px e 320px.

Mobile-first vs Desktop-first

Mobile-first: você escreve os estilos base para telas pequenas e usa min-width para adicionar estilos conforme a tela cresce.
Desktop-first: o contrário — estilos base pra tela grande, max-width para adaptar em telas menores.
O P5 parte de um design desktop, então desktop-first costuma ser mais direto aqui.

Desktop-first — exemplo
.lead__title {
  font-size: 56px;     /* desktop: 1280px */
}

@media (max-width: 768px) {
  .lead__title {
    font-size: 40px;   /* tablet */
  }
}

@media (max-width: 320px) {
  .lead__title {
    font-size: 28px;   /* mobile */
  }
}
💡 sobre o checklist
O checklist permite até 30px de diferença em 1280px e até 10px em 768px e 320px. Isso significa que 768px e mobile são mais exigentes — dê atenção especial a eles.
⚠ armadilha clássica
Nunca duplique declarações entre media queries. Se .section tem padding: 80px no desktop e padding: 40px no mobile, coloque só a diferença dentro do @media, não o bloco inteiro. O checklist rejeita duplicação entre at-rules.
05 /

Media Queries

Media queries são a ferramenta que torna o CSS responsivo. Elas aplicam estilos condicionalmente, dependendo das características da tela.

Sintaxe básica
@media (max-width: 768px) {
  /* estilos aplicados quando a tela tem até 768px */
  .container {
    padding: 0 16px;
  }
}

@media (min-width: 769px) {
  /* estilos aplicados quando a tela tem pelo menos 769px */
}
Demo interativa — redimensione a caixa arrastando o canto inferior direito
Este bloco muda de cor abaixo de 480px

azul = acima de 480px  ·  vermelho = abaixo de 480px

Como esse demo funciona
.mq-card {
  background: blue;
}

@media (max-width: 480px) {
  .mq-card {
    background: red;  /* só a diferença */
  }
}

A ordem das media queries importa

No desktop-first, as media queries com max-width menores devem vir depois no arquivo — elas sobrescrevem estilos anteriores. Se você colocar max-width: 320px antes de max-width: 768px, o 768px vai sobrescrever o 320px em algumas resoluções.

pergunta pra reflexão

Você tem .section { padding: 80px 40px } e precisa que em 768px o padding seja 48px 24px e em 320px seja 32px 16px. Quantas declarações CSS você precisa no total? Onde cada uma vai?

Checklist do Projeto 5

Os pontos de rejeição imediata que você precisa garantir antes de submeter:

  • Media queries implementadas
    sem @media = rejeição automática
  • Layout responsivo funciona em 1280px, 768px e 320px
    tolerância: 30px em 1280px · 10px em 768px e 320px
  • Nenhum texto ultrapassa o bloco pai
    overflow de texto é rejeição imediata
  • Sem scrollbar horizontal
    teste redimensionando a janela devagar
  • Elementos de texto sem altura fixa
    cinco ou mais elementos com height fixo = rejeição
  • Sem duplicação de estilos entre media queries
    cada @media só declara o que muda
  • BEM correto — sem números nos nomes de classe
    block__element1 ou block__element-one são erros
  • max-width definido para o conteúdo
    de acordo com o design
  • Fonte Inter configurada com alternativas de sistema
    font-family: 'Inter', Arial, sans-serif
💡 fluxo de revisão
Antes de submeter: abre o DevTools, vai em responsive mode e arrasta a largura de 1280px até 320px devagar. Se o layout quebrar em qualquer ponto, você encontrou um problema que a revisão também vai encontrar.
